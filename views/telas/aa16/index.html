Nova Estrutura de
Acessos SAP
Os acessos SAP são desenhados de acordo com boas práticas de segurança da informação para facilitar a administração e simplificar a gestão de acessos. No Grupo Petrópolis, foram adotados alguns conceitos para facilitar o entendimento e a aplicação.

TRANSAÇÃO
Código de acesso a telas para realizar determinada atividade no sistema SAP.
OBJETO DE AUTORIZAÇÃO
Efetua o controle da transação para restrição e permissão das microatividades, por exemplo, criar lançamento para tipos de documentos, movimentos, entre outras restrições possíveis.
NÍVEL ORGANIZACIONAL
Permissões concedidas de acordo com as restrições de regras de negócio. Têm referência direta com a unidade organizacional, o centro de custos, a subárea de RH etc.
MICROATIVIDADE
Conjunto de transações agrupadas por atividade, por exemplo:
  • microatividade 1: criar pedidos de compras;
  • microatividade 2: exibir pedidos de compras.
FUNÇÃO
Agrupamento de microatividades relacionadas com as atribuições dos colaboradores executadas no sistema SAP, que devem estar alinhadas à rotina de trabalho.
